## What DIGIMON Is

A composable GraphRAG system: 28 typed operators composed at runtime into retrieval DAGs
by an LLM agent. The agent builds and queries knowledge graphs from document corpora and
answers multi-hop factual questions. Thesis: adaptive operator routing outperforms fixed
pipelines across question types.

**Active benchmark**: MuSiQue (multi-hop QA). **Primary metric**: LLM-judge (llm_em).

**Key architecture**:
- `Core/MCP/tool_consolidation.py` — 28 operators exposed as 10 consolidated tools via MCP
- `eval/run_agent_benchmark.py` — benchmark runner (MCP or direct-Python backend)
- `prompts/agent_benchmark_consolidated.yaml` — agent system prompt (Jinja2 template)
- `digimon_mcp_stdio_server.py` — MCP server exposing all operators

**Running a benchmark**:
```bash

make sentinel               # 2 stable questions for regression check (~$0.12, ~2 min)

### 2. IEE Family Fix (HIGHEST IMPACT, ~4/6 failing questions)

**Problem**: Agent resolves intermediate entity to wrong semantic type.
- "holy family" (a group) when question says "the person who"
- "Middle East" (macro-region) when question needs named sub-region
- "Joseph Smith" when question needs "Joseph of Nazareth"
- "Adelphia" when question needs the successor (Time Warner Cable)

**Root cause**: `entity_search` returns multiple candidates; agent picks wrong one.
No verification that the chosen entity's semantic type satisfies the question constraint.

**Approaches to consider**:
a. **Disambiguation step at entity selection time**: After entity_search, before committing
   to an entity, verify it matches the question's implicit type constraint by checking
   chunk_retrieve for the specific relationship. Step 3d in the current prompt already
   says this — but agents often skip it. Make it more explicit.
b. **Scoped metacognitive check**: When the question contains "the person who", "the
   organization that", "the specific [X]" — after resolving an intermediate entity, add
   one required self-check: "does this entity's type match the constraint?" This is a
   narrower version of the doubt field (which was too broad).
c. **entity_search improvement**: Add explicit guidance on how to use type constraints
   in the search query (e.g., "Joseph of Nazareth saint" vs just "Joseph").

### 3. 619265 (Batman Beyond, "12") — Season-Specific Retrieval

→ submits "52". Gold answer is "12" (season 5 episodes specifically).

```

instead of relying on the entity description. The prompt already says chunk_retrieve
is mandatory — but the agent's query isn't specific enough.

General heuristic to add: "When the question asks about a SPECIFIC SUBSET (season N,
year Y, one event), the entity description may have the aggregate — always query

## Important Constraints

1. **All fixes must be general** — no question-specific patches. Fix the failure FAMILY
   (e.g., "IEE where entity type mismatches question constraint"), not one question.

2. **Stochasticity policy** — ≥2 runs showing same result, OR ≥3 question net improvement,
   before claiming a fix is real. Single-run single-question flips are noise.

3. **Never truncate evidence text** — `CLAUDE.md` rule. Entity descriptions, chunk text,
   relationship descriptions must be shown in full to the agent. Linearization summarizes
   metadata (IDs, scores, counts) but never truncates actual evidence.

## Graph Data Context

The MuSiQue graph (`results/MuSiQue/er_graph/nx_data.graphml`) is a pre-built ER graph:
- ~5,000 entity nodes with `entity_name`, `entity_type`, `description` attributes
- Entity descriptions are synthesized from multiple source chunks — they contain attribute
  facts (dates, counts, qualifications) but as AGGREGATES, not specific sub-facts
- ~30,000+ relationship edges with `relation_name` and `description`
- Passage nodes added (HippoRAG v2 bipartite)

Venice example (relevant for 820301):
- Node: `venice`, type=geo, description includes "22 plague outbreaks between 1361 and 1528"
- Edge: `venice → plague in venice 1361 1528`, description "venice experienced 22 plague occurrences"
- Corpus chunk 267: "Plague occurred in Venice 22 times between 1361 and 1528"

Batman Beyond example (relevant for 619265):
- Node: `batman beyond`, type=show, description includes "52 total episodes" (aggregate)
- Season 5 has 13 episodes (actually 13, not 12 — gold may have an error or count differently)
- Corpus has season-specific episode counts in separate chunks

## Quick Diagnostic Recipe

When a question fails, check in this order:

```bash
# 1. Is the answer in the corpus?
python -c "
import json
corpus = json.load(open('results/MuSiQue/corpus/Corpus.json'))
for chunk in corpus['corpus']:
    if 'SEARCH_TERM' in chunk['content'].lower():
        print(chunk['chunk_id'], chunk['content'][:200])
"

# 2. Is the answer in the graph?
python -c "
import networkx as nx
G = nx.read_graphml('results/MuSiQue/er_graph/nx_data.graphml')
for n, d in G.nodes(data=True):
    if 'ENTITY_NAME' in str(d.get('entity_name','')).lower():
        print(n, d.get('description','')[:300])
"

# 3. What did the agent actually do?
python scripts/diagnose_question.py results/MuSiQue_gpt-5-4-mini_consolidated_XXXXZ.json QID

# 4. Timing breakdown for that question
python scripts/timing_report.py --trace digimon.benchmark.MuSiQue.QID
```
